# Teru Credit & Victoria 111 — static websites

Two white-label static marketing websites built from **one Angular 21 codebase**.
Both sites share the same layout and components (design based on
[itfgroupbg.com](https://itfgroupbg.com/)); each brand differs only by its
runtime configuration, images and translation overlays.

## The white-label pattern

There is **one build**. The active brand is chosen by swapping
`public/config.json`, which the app fetches at startup before it renders:

- **Brand identity** (name, theme colours, contact, feature flags, key figures,
  product/image paths) lives in `public/brands/config.<brand>.json`. At runtime
  `ConfigService` loads it and writes the theme onto `:root` as CSS custom
  properties (`--color-accent`, …). Components read those variables — never
  hard-coded colours or `brand === 'x'` checks. Sections are toggled by the
  `features.*` booleans in the config.
- **Text content** is bilingual (Bulgarian / English). A shared base dictionary
  (`public/i18n/{bg,en}.json`) is deep-merged with a per-brand overlay
  (`public/i18n/<brand>/{bg,en}.json`) by `I18nService`, so brand-specific copy
  overrides the base while everything common stays in one place.
- **Images** live in `public/images/<brand>/` (currently generated SVG
  placeholders — replace with real assets).

## Commands

```bash
# Develop a brand (serves at http://localhost:4200)
npm run start:victoria
npm run start:teru

# Build a brand → dist/<brand>/browser/  (the deployable static folder)
npm run build:victoria
npm run build:teru
npm run build:all        # both

# Just switch the active brand's config.json without building
npm run config:victoria
npm run config:teru

# Regenerate placeholder images
npm run gen:images
```

Each `build:*` output (`dist/victoria/browser`, `dist/teru/browser`) is a fully
static site that can be hosted on any static host.

## Project structure

```
public/
  config.json                # active brand (generated by set-brand.mjs)
  brands/config.<brand>.json # per-brand identity/theme/features
  i18n/{bg,en}.json          # shared base copy
  i18n/<brand>/{bg,en}.json  # per-brand overlay
  images/<brand>/            # per-brand images
src/app/
  core/                      # ConfigService, I18nService, brand model
  layout/                    # header (nav + language toggle), footer
  sections/                  # hero, about, mission, competencies,
                             #   products, key-figures, news, contacts
  pages/home/                # assembles the sections per feature flags
scripts/
  set-brand.mjs              # copies config.<brand>.json → config.json
  gen-placeholders.mjs       # generates placeholder SVG images
```

## Adding or editing a brand

1. Copy `public/brands/config.victoria.json` to `config.<brand>.json` and edit
   the name, theme, contact, key figures and image paths.
2. Add `public/i18n/<brand>/{bg,en}.json` with any copy you want to override.
3. Drop real images into `public/images/<brand>/`.
4. Add `start:<brand>` / `build:<brand>` scripts mirroring the existing ones.
